사용자가 알려준 테스트 케이스 시나리오를 바탕으로 아래 형식에 따라 각 테스트 케이스 제안 후 사용자와의 논의를 통해 테스트 케이스 내용을 결정해줘
테스트 케이스 시나리오를 모르는 상황이면 알려달라고 해

## 1. 헤더 라인

```
**TC-X.X.X: [테스트 이름] 🟢/🟨/🔴 Happy/Edge/Error 🔵 Unit/Int/E2E**
```

- **TC-X.X.X**: 테스트 케이스 번호 (예: TC 1.6.1)
- **[테스트 이름]**: 간단한 테스트 설명
- **상태 아이콘**:
  - 🟢 Happy Path (정상 동작)
  - 🟨 Edge Case (경계 케이스)
  - 🔴 Error Case (에러 처리)
- **타입 아이콘**:
  - 🔵 Unit (단위 테스트)
  - 🟠 Integration (통합 테스트)
  - 🟣 E2E (엔드투엔드 테스트)

---

## 2. Given-When-Then 섹션

### **Given (주어진 상태)**
- 테스트 시작 전 초기 상태
- 필요한 모든 전제 조건
- 의존성 및 환경 설정

### **When (실행 동작)**
- 사용자가 수행하는 액션
- 입력 데이터와 파라미터
- 단계별 수행 순서

### **Then (예상 결과)**
- 액션 후 예상되는 결과
- 검증해야 할 항목
- 성공 기준

---

## 3. 공통 설정

- **환경**: 테스트에 필요한 도구, 라이브러리
- **각 테스트 전 (Setup)**: 테스트 실행 전 준비 작업
- **각 테스트 후 (Teardown)**: 테스트 완료 후 정리 작업
